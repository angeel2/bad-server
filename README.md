## Выполненные исправления безопасности

### 1. XSS (Межсайтовый скриптинг)
- Добавлена функция `escapeHtml()` для экранирования спецсимволов
- Очистка пользовательского ввода в полях: комментарий, телефон, email, адрес
- Удалены `dangerouslySetInnerHTML` во фронтенде

### 2. CSRF (Межсайтовая подделка запросов)
- Добавлен middleware для проверки origin/referer заголовков
- Защищены все изменяющие данные маршруты (POST, PATCH, DELETE)

### 3. NoSQL-инъекции
- Принудительное преобразование входящих данных в строку
- Фильтрация опасных операторов MongoDB (`$ne`, `$gt`, `$regex` и др.)
- Валидация ObjectId перед запросом к БД

### 4. Переполнение буфера
- Ограничение размера JSON запросов до 1MB
- Ограничение размера загружаемых файлов до 5MB
- Ограничение количества товаров в заказе до 100

### 5. ReDoS (Regular Expression Denial of Service)
- Создана функция `safeRegex()` с ограничением длины строки (100 символов)
- Экранирование спецсимволов в регулярных выражениях

### 6. DDoS (Rate Limiting)
- Добавлен `express-rate-limit` для всех запросов (100 запросов/15 минут)
- Строгие лимиты для аутентификации (5 попыток/15 минут)
- Лимиты для заказов (20 заказов/час) и загрузки файлов (10/час)

### 7. Path Traversal (Обход директорий)
- Валидация путей к файлам
- Запрет символов `..` и абсолютных путей
- Нормализация путей перед доступом

### 8. Кэширование
- Добавлено кэширование для часто запрашиваемых ресурсов (GET /product)
- TTL кэша: 1 минута
- Автоматический сброс кэша при изменении данных

### 9. Аудит зависимостей
- Выполнена команда `npm audit fix`
- Устранены все критические уязвимости

### 10. Линтинг и типизация
- Исправлены все ошибки ESLint
- Устранены проблемы TypeScript
- Код приведён к стандартам
